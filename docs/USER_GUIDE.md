# 用户使用指南：Web3 Data MCP 提示词增强功能

## 🎯 简介

Web3 Data MCP 服务器现已集成智能提示词增强功能，让您与 Web3 数据的交互更加准确、高效。本指南将帮助您充分利用这些增强功能。

## 🚀 快速开始

### 在 Claude Desktop 中使用

1. **确保服务已启动**
   - 打开 Claude Desktop
   - 检查 MCP 服务器状态（应显示 "web3-data-mcp" 为活动状态）

2. **开始查询**
   - 直接用自然语言提问
   - 系统会自动理解您的意图并选择合适的工具

### 基础查询示例

```
// 搜索项目
"搜索以太坊上的 DeFi 项目"
"find GameFi projects"
"查找 Layer2 解决方案"

// 获取项目详情
"获取 Uniswap 的详细信息"
"show me details about project 11646"
"Aave 项目的最新数据"

// 检查积分
"查看我的积分余额"
"check my remaining credits"
```

## 📝 增强功能特性

### 1. 智能意图理解

系统现在能更准确地理解您的查询意图：

| 您说的 | 系统理解为 |
|-------|-----------|
| "uni 项目" | 搜索 Uniswap 相关项目 |
| "找 defi" | 搜索 DeFi 类别的项目 |
| "项目 12345" | 获取 ID 为 12345 的项目详情 |
| "eth 生态" | 搜索 Ethereum 生态系统项目 |

### 2. 缩写自动扩展

常用缩写会自动扩展为完整名称：

- `uni` → `Uniswap`
- `comp` → `Compound`
- `aave` → `Aave`
- `defi` → `DeFi`
- `gamefi` → `GameFi`
- `btc` → `Bitcoin`
- `eth` → `Ethereum`

### 3. 多语言支持

支持中英文混合查询：

```
// 英文
"search for DeFi protocols on Ethereum"

// 中文
"搜索以太坊上的 DeFi 协议"

// 混合
"查找 Ethereum 的 DeFi projects"
```

### 4. 智能响应格式

#### 搜索结果增强

当您搜索项目时，响应会包含：

```json
{
  "data": [...],
  "interpretation": {
    "summary": "找到 5 个以太坊上的 DeFi 项目"
  },
  "suggestions": [
    {
      "description": "获取 Uniswap 的详细信息",
      "action": "运行：get_project_details(11646)"
    }
  ],
  "dataQuality": {
    "level": "high",
    "indicators": ["数据完整", "最近更新"]
  }
}
```

#### 空结果帮助

如果没有找到结果，系统会提供帮助：

```json
{
  "emptyResultHelp": {
    "message": "未找到匹配的项目",
    "suggestions": [
      "尝试使用更通用的搜索词",
      "检查项目名称拼写",
      "使用不同的关键词"
    ],
    "alternativeQueries": [
      "search DeFi projects",
      "find Ethereum projects"
    ]
  }
}
```

## 🛠️ 高级用法

### 1. 精确搜索

使用特定格式获得更精确的结果：

```
// 按项目 ID 搜索
```

组合多个条件：

```
// 生态系统 + 类别
"Ethereum 上的 DeFi 项目"
"Solana GameFi projects"

// 项目 + 详情类型
"Uniswap 的团队信息"
"Aave funding rounds"
```

### 3. 批量操作建议

当查询多个项目时，系统会建议批量操作：

```
输入："比较 Uniswap、Aave 和 Compound"

系统建议：
1. 先搜索这三个项目
2. 分别获取详情
3. 整理对比数据
```

## ❓ 常见问题

### Q1: 为什么有时候搜索结果为空？

**可能原因：**
1. 搜索词太具体
2. 项目名称拼写错误
3. 使用了不常见的缩写

**解决方法：**
- 使用更通用的搜索词
- 检查拼写
- 尝试完整的项目名称

### Q2: 如何知道剩余积分？

直接询问：
- "查看积分"
- "check credits"
- "剩余配额"

### Q3: 支持哪些查询语言？

目前支持：
- 英文（en）
- 中文（zh）
- 中英混合

### Q4: 如何获取最准确的结果？

**最佳实践：**
1. 使用完整的项目名称
2. 指定具体的查询条件
3. 使用项目 ID（如果知道）
4. 明确说明需要的信息类型

## 💡 使用技巧

### 1. 利用智能建议

每次查询后，注意系统提供的建议操作，这些建议基于您的查询上下文生成。

### 2. 渐进式查询

```
第一步：广泛搜索
"搜索 DeFi 项目"

第二步：缩小范围
"在结果中找借贷协议"

第三步：获取详情
"Aave 的详细信息"
```

### 3. 使用实体识别

系统能识别多种实体类型：
- 项目名称
- 代币符号
- 生态系统
- 合约地址
- 数字 ID
- Twitter 账号
- 组织名称

### 4. 查看数据质量

每个响应都包含数据质量指标：
- `high`：数据完整且最新
- `medium`：数据基本完整
- `low`：数据可能不完整或过时

## 🔧 故障排除

### 错误：积分不足

```json
{
  "error": {
    "message": "积分不足，需要 10 个，当前只有 5 个",
    "suggestion": "请充值账户或使用需要较少积分的功能",
    "alternativeTools": ["免费搜索功能"]
  }
}
```

**解决方案：**
- 使用建议的替代工具
- 联系管理员充值
- 优化查询减少积分消耗

### 错误：参数无效

```json
{
  "error": {
    "message": "项目 ID 格式无效",
    "suggestion": "请使用数字 ID 或有效的合约地址",
    "examples": ["12345", "0x1f9840a85d5af5bf1d1762f925bdaddc4201f984"]
  }
}
```

**解决方案：**
- 按照建议的格式重新输入
- 参考提供的示例

## 📊 性能提示

1. **缓存机制**
   - 相同查询会使用缓存结果
   - 缓存有效期为 30 分钟
   - 实时数据请求会绕过缓存

2. **批量查询**
   - 同时查询多个项目更高效
   - 系统会自动优化批量请求

3. **响应时间**
   - 一般查询：< 100ms
   - 复杂查询：< 500ms
   - 首次查询可能稍慢（缓存预热）
